# Руководство по редактированию HTML файлов

## Введение

Этот проект использует систему шаблонов Django для создания HTML страниц. Django шаблоны позволяют создавать переиспользуемые компоненты и структурировать код более эффективно.

## Как работает система шаблонов Django

### Базовый шаблон (base.html)

В проекте есть базовый шаблон `public/base.html`, который содержит общую структуру всех страниц:
- HTML структуру (DOCTYPE, html, head, body)
- Навигационное меню
- Футер
- Общие CSS и JavaScript файлы

### Расширение базового шаблона

Все остальные HTML файлы расширяют базовый шаблон с помощью директивы `{% extends "base.html" %}`:

```django
{% extends "base.html" %}
```

Эта директива должна быть **первой строкой** в файле.

## Блоки (Blocks)

### Что такое блоки?

Блоки - это специальные области в шаблоне, которые можно переопределить в дочерних шаблонах. Они позволяют изменять содержимое определенных частей страницы без дублирования всего кода.

### Синтаксис блоков

Блоки определяются с помощью следующих тегов:

```django
{% block название_блока %}
    Содержимое блока
{% endblock %}
```

### Основные блоки в проекте

#### 1. Блок `title`
Определяет заголовок страницы (отображается во вкладке браузера):

```django
{% block title %}Заголовок страницы{% endblock %}
```

**Пример из index.html:**
```django
{% block title %}Главная - Приложение Taskboard{% endblock %}
```

#### 2. Блок `content`
Основное содержимое страницы. Это самый важный блок, который содержит весь контент страницы:

```django
{% block content %}
    <div class="hero">
        <h1>Добро пожаловать</h1>
        <!-- Ваш HTML код здесь -->
    </div>
{% endblock %}
```

**Пример из index.html (строки 5-49):**
```django
{% block content %}
<div class="hero">
    <div class="container">
        <h1>Добро пожаловать в приложение Taskboard</h1>
        <!-- ... остальной контент ... -->
    </div>
</div>
{% endblock %}
```

#### 3. Блок `extra_css`
Используется для подключения дополнительных CSS файлов, специфичных для конкретной страницы:

```django
{% block extra_css %}
<link rel="stylesheet" href="/static/css/auth.css">
{% endblock %}
```

**Пример из login.html:**
```django
{% block extra_css %}
<link rel="stylesheet" href="/static/css/auth.css">
{% endblock %}
```

#### 4. Блок `extra_js`
Используется для подключения дополнительных JavaScript файлов и скриптов:

```django
{% block extra_js %}
<script src="/static/js/auth.js"></script>
<script>
    // Ваш JavaScript код здесь
</script>
{% endblock %}
```

**Пример из index.html (строки 51-59):**
```django
{% block extra_js %}
<script src="/static/js/public-tasks.js"></script>
<script>
    document.addEventListener('DOMContentLoaded', function() {
        loadPublicTasks();
    });
</script>
{% endblock %}
```

## Как редактировать HTML файлы

### Шаг 1: Выберите файл для редактирования

Все HTML файлы находятся в директории `public/`:
- `public/index.html` - главная страница
- `public/accounts/login.html` - страница входа
- `public/accounts/register.html` - страница регистрации
- `public/profile/index.html` - страница профиля
- `public/task/index.html` - страница со списком задач
- `public/base.html` - базовый шаблон (редактируйте осторожно!)

### Шаг 2: Определите, какой блок нужно изменить

- **Для изменения заголовка страницы** - редактируйте блок `{% block title %}`
- **Для изменения основного содержимого** - редактируйте блок `{% block content %}`
- **Для добавления CSS** - редактируйте блок `{% block extra_css %}`
- **Для добавления JavaScript** - редактируйте блок `{% block extra_js %}`

### Шаг 3: Внесите изменения

**Важно:** Всегда сохраняйте структуру блоков:
- Начало блока: `{% block название %}`
- Конец блока: `{% endblock %}`

**Пример редактирования контента:**

Допустим, вы хотите изменить текст на главной странице. Откройте `public/index.html` и найдите блок `content`:

```django
{% block content %}
<div class="hero">
    <div class="container">
        <h1>Добро пожаловать в приложение Taskboard</h1>
        <!-- Измените текст здесь -->
        <p class="hero-subtitle">Новый текст подзаголовка</p>
    </div>
</div>
{% endblock %}
```

### Шаг 4: Проверьте синтаксис

Убедитесь, что:
- ✅ Каждый `{% block %}` имеет соответствующий `{% endblock %}`
- ✅ Блоки правильно вложены (не пересекаются)
- ✅ HTML теги правильно закрыты
- ✅ Нет опечаток в названиях блоков

## Примеры использования

### Пример 1: Добавление нового раздела на страницу

Чтобы добавить новый раздел на главную страницу:

```django
{% block content %}
<div class="hero">
    <!-- Существующий контент -->
</div>

<!-- Новый раздел -->
<div class="new-section">
    <div class="container">
        <h2>Новый раздел</h2>
        <p>Содержимое нового раздела</p>
    </div>
</div>
{% endblock %}
```

### Пример 2: Добавление дополнительного CSS

Чтобы добавить стили для конкретной страницы:

```django
{% block extra_css %}
<link rel="stylesheet" href="/static/css/auth.css">
<style>
    .custom-class {
        color: blue;
    }
</style>
{% endblock %}
```

### Пример 3: Добавление JavaScript функциональности

Чтобы добавить JavaScript код:

```django
{% block extra_js %}
<script src="/static/js/auth.js"></script>
<script>
    document.addEventListener('DOMContentLoaded', function() {
        // Ваш код здесь
        console.log('Страница загружена');
    });
</script>
{% endblock %}
```

## Важные замечания

### ⚠️ Не удаляйте блоки!

Каждый файл должен содержать все необходимые блоки. Если вы удалите `{% endblock %}`, страница не будет работать правильно.

### ⚠️ Порядок блоков важен

Хотя порядок блоков не критичен для Django, рекомендуется следовать такой структуре:

1. `{% block title %}`
2. `{% block extra_css %}`
3. `{% block content %}`
4. `{% block extra_js %}`

### ⚠️ Не редактируйте base.html без необходимости

Базовый шаблон `base.html` влияет на все страницы. Изменения в нем затронут весь сайт.

## Структура типичного HTML файла

```django
{% extends "base.html" %}

{% block title %}Название страницы{% endblock %}

{% block extra_css %}
<!-- Дополнительные CSS файлы -->
{% endblock %}

{% block content %}
<!-- Основное содержимое страницы -->
<div class="container">
    <h1>Заголовок</h1>
    <p>Текст</p>
</div>
{% endblock %}

{% block extra_js %}
<!-- Дополнительные JavaScript файлы и скрипты -->
<script>
    // JavaScript код
</script>
{% endblock %}
```

## Часто задаваемые вопросы

### Q: Что произойдет, если я забуду закрыть блок `{% endblock %}`?
A: Django выдаст ошибку при рендеринге шаблона. Всегда проверяйте, что каждый `{% block %}` имеет соответствующий `{% endblock %}`.

### Q: Могу ли я использовать несколько блоков с одинаковым именем?
A: Нет, в одном файле не может быть двух блоков с одинаковым именем. Последний блок перезапишет предыдущий.

### Q: Можно ли вложить один блок в другой?
A: Нет, блоки не могут быть вложены друг в друга. Но вы можете использовать обычные HTML теги внутри блоков.

### Q: Как изменить навигационное меню?
A: Навигационное меню находится в `base.html`. Отредактируйте файл `public/base.html`, но помните, что изменения затронут все страницы.
